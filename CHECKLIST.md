## 2-Week Checklist (14 Days)

### Week 1
- [X] Day 1: Define goal + success criteria (what “done” means)
- [X] Day 2: Break work into tasks + estimate effort
- [X] Day 3: Set up repo structure (folders, naming, docs skeleton) - Day 1 - 3 : Automated
- [X] Day 4: Build baseline / first working version (even ugly)
    - Problem: Bot can only scan 1 link at atime, 
    - Mini Win: Bot is working good so far

- [ ] Day 5: Core feature 1 implemented
- [ ] Day 6: Core feature 2 implemented
- [ ] Day 7: Review week + fix top 3 problems

### Week 2
- [ ] Day 8: Add logging/monitoring (or basic debugging helpers)
- [ ] Day 9: Improve reliability (edge cases, error handling)
- [ ] Day 10: Write tests (minimum: critical paths)
- [ ] Day 11: Refactor cleanup (remove hacks, rename things properly)
- [ ] Day 12: Documentation (setup, usage, screenshots/examples)
- [ ] Day 13: Final testing + performance check
- [ ] Day 14: Release/submit (tag, changelog, final report)

---

## Daily mini-checklist (repeat each day)
- [ ] Pick 1–3 priority tasks for today
- [ ] Do the hardest task first (before distractions win)
- [ ] Commit progress (even small)
- [ ] Write 1–2 lines of notes: what changed + what’s next
- [ ] Quick review: what blocked me, how to remove it tomorrow
